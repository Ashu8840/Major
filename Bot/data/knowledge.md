### Question

Data Security

### Answer

All data is transmitted over HTTPS and passwords are hashed with bcrypt. JWT tokens secure your session without storing credentials in the database.

---

### Question

JWT Authentication

### Answer

SoulSpace uses JSON Web Tokens for authentication. Your token is stored in your browser and sent with each request automatically.

---

### Question

Session Expiry

### Answer

JWT tokens expire after a set period. When your token expires, you will be automatically redirected to the login page.

---

### Question

Password Security

### Answer

Passwords are hashed with bcrypt using 10 salt rounds before storage. Your plain-text password is never saved in the database.

---

### Question

Input Sanitization

### Answer

All user inputs are sanitized to prevent script injection and NoSQL injection attacks, protecting the platform and all users.

---

### Question

Rate Limiting

### Answer

SoulSpace limits requests to 100 per 15 minutes per IP address to prevent abuse and protect server resources.

---

### Question

CORS Protection

### Answer

Only authorized origins (frontend domain) can make requests to the SoulSpace backend API, preventing cross-site request attacks.

Pair programming sessions have been used during complex feature development like nested comments and real-time chat.

### Question

Documentation Standards

### Answer

All API endpoints are documented with request/response examples. All React components have PropTypes defined for type safety.

---

### Question

Naming Conventions

### Answer

camelCase is used for JavaScript variables and functions, PascalCase for React components, and kebab-case for CSS class names.

---

### Question

Folder Organization

### Answer

Frontend follows feature-based folder organization. Backend follows MVC (Models, Views, Controllers) architectural pattern.

---

### Question

Git Commit Messages

### Answer

Commit messages follow the Conventional Commits standard (feat:, fix:, docs:, refactor:) for clear history and changelog generation.

---

### Question

Branch Strategy

### Answer

Feature branches are created from main, named feat/feature-name or fix/bug-description, and deleted after merging.

---

### Question

Hotfix Process

### Answer

Critical production bugs are fixed in hotfix branches branched from main, immediately merged and deployed.

---

### Question

Release Tagging

### Answer

Releases are tagged in Git using semantic versioning (v1.0.0) for clear version history tracking.

Project Roadmap

The SoulSpace roadmap prioritizes: (1) Testing & stability, (2) PWA & offline support, (3) Stories feature, (4) Marketplace, (5) Multi-language support.

Known Limitations
